# 친절한 SQL 튜닝
## 2장 인덱스 기본

## 2.1 인덱스 구조 및 탐색
테이블에서 데이터를 찾는 방법
- 테이블 전체를 스캔한다.
- 인덱스를 이용한다.

인덱스는 큰 테이블에서 소량 데이터를 검색할 때 사용한다.

### 인덱스 튜닝의 두 가지 핵심요소
- 인덱스 스캔 효율화 튜닝 : 스캔 과정에서 발생하는 비효율을 줄이는 것
- **랜덤 액세스 최소화 튜닝** : 테이블 액세스 횟수를 줄이는 것

!! SQL 튜닝은 랜덤 I/O와의 전쟁이다.

인덱스란 ?
대용량 테이블에서 필요한 데이터만 빠르게 효율적으로 액세스하기 위해 사용하는 **오브젝트**
책의 목차와 같다.

인덱스를 이용하면 Range Scan(범위 스캔)이 가능하다. (정렬되어있기 때문에 가능!)

DBMS는 일반적으로 B*Tree 인덱스를 사용한다.

LMC(Leftmost Child) : 자식 노드 중 가장 왼쪽 블록을 가르킨다.
리프 블록 : 키값으로 정렬되어 있고 RowID(주소값)을 가진다.
- ROWID = 데이터 블록 주소 + 로우번호
- 데이터 블록 주소 = 데이터 파일 번호 + 블록 번호
- 블록 번호 : 데이터 파일 내에서 부여한 상대적 순번
- 로우 번호 : 블록 내 순번

<인덱스 탐색 과정>
- 수직적 탐색 : 인덱스 스캔 시작지점을 찾는 과정
- 수평적 탐색 : 데이터를 찾는 과정

결합인덱스는 두개 이상의 컬럼을 결합해서 만든 인덱스.
-> 선택도가 낮은 컬럼ㅇ르 앞에 두고 인덱스를 생성하는 것이 유리하다.

## 2.2 인덱스 기본 사용법
== 인덱스를 Range Scan 하는 방법

**인덱스 컬럼을 가공하면 Range Scan할 수 없다.**
LIKE, OR, IN 조건도 불가하다.

```oracle
where (전화번호 = :tel_no OR 고객명 = :cust_nm)

where 전화번호 in (:tel_no1, :tel_no2)
```
IN의 경우 UNION ALL로 변경하면 가능할 수도 있다. 
이 경우 옵티마이저가 쿼리 변환을 통해 Range Scan으로 처리되기도 한다.
```oracle
select *
from 고객
where 전화번호 = :tel_no1
union all
select *
from 고객
where 전화번호 = :tel_no2
```

**인덱스 선두 컬럼이 조건절에 있어야한다.**

인덱스를 잘 탄다고 해서 튜닝이 끝이 아니다.
1. 실제 데이터 특성을 잘 따져보아야 한다. (인덱스 리프 블록에서 스캔하는 양을 고려해야한다.)
2. 데이터 값의 특성이 계속 해서 변할 수 있기 때문 (카디널리티가 계속 변화한다.)


인덱스를 이용한 소트 연산 생략
**인덱스는 정렬되어있기 때문에 인덱스를 이용해서 정렬 효과를 부수적으로 얻을 수 있다.**
